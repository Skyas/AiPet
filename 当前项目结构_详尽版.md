## 总体架构

```
┌──────────────────────────────────────────────────────┐
│             Electron 桌面前端（渲染层）                │
│   ChatPanel / VisionPanel / SettingsPanel / PromptEditor │
│   Pinia Store ← utils/api.js (REST+SSE) ← axios      │
│   Socket.IO 单例 (utils/socket.js) ← 实时推送        │
└───────────────────┬──────────────────────────────────┘
                    │ HTTP REST + SSE + WebSocket
                    │ localhost:8001
┌───────────────────▼──────────────────────────────────┐
│              Python FastAPI 后端（业务层）             │
│                                                      │
│  ┌──────────────────────────────────────────────┐   │
│  │ 核心模块（始终运行）                           │   │
│  │  ai_chat · ai_vision · screen_capture        │   │
│  │  history · vision_memory · proactive_engine  │   │
│  └──────────────────────────────────────────────┘   │
│                                                      │
│  ┌─────────────────────┐  ┌────────────────────┐    │
│  │  语音模块 🔘 可开关  │  │  QQ模块 🔘 可开关  │    │
│  │  （Phase 3 占位）   │  │  （Phase 4 占位）   │    │
│  └─────────────────────┘  └────────────────────┘    │
└──────────────────────────────────────────────────────┘
```

**说明：**
- 前端通过 `axios` 调用 REST API，通过 `fetch` 消费 SSE 流式响应
- 后端通过 `python-socketio` 向前端推送主动消息（`proactive_message` 事件）
- 语音模块与 QQ 模块默认关闭，关闭时对应依赖不加载，不占用端口

### 入口与配置

#### `main.py` — 应用入口

**职责：**
- 创建 FastAPI 实例，注册 CORS 中间件
- 注册全部路由：`chat_router` / `settings_router` / `prompts_router` / `screen_router`
- 初始化 `app.state`：`config` / `sio`（Socket.IO）/ `chat_module` / `vision_module` / `vision_memory` / `proactive_engine`
- 使用 `lifespan` 上下文管理启动/关闭：若配置中 `proactive_enabled=True` 则自动启动主动引擎
- 包装为 `socketio.ASGIApp`，uvicorn 监听 `0.0.0.0:8001`

**Socket.IO 事件：**
- `connect` → 向客户端推送 `server_info`（版本号 + 模块状态）
- `disconnect` → 打印日志

**健康检查：** `GET /health` → 返回状态、版本、主动引擎运行状态

### API 路由层

#### `api/chat.py` — 对话路由

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/chat/send` | POST | SSE 流式对话（支持 Vision 注入） |
| `/api/chat/history/{session_id}` | GET | 获取历史消息（`limit` 参数） |
| `/api/chat/history/{session_id}` | DELETE | 清空历史 |
| `/api/chat/sessions` | GET | 列出所有会话 |

**`/api/chat/send` 核心流程：**
1. 通知 `proactive_engine` 用户活跃（重置冷却计时器）
2. 保存用户消息到 SQLite
3. 若 `vision.enabled && inject_on_chat`，并行发起截图任务（`asyncio.create_task`）
4. 获取历史（最近 20 条）+ 对应 Prompt 卡片
5. 等待截图结果，按视觉路由决定注入方式：
   - `multimodal` 路由 → 将图片 Base64 嵌入最后一条 user 消息（多模态格式）
   - 其他路由 → 将视觉描述文字追加到消息内容
6. 从 `vision_memory` 读取屏幕变化摘要，注入 `system_prompt`
7. 流式调用 `AIChatModule.chat_stream()`，逐 token 以 SSE 格式返回
8. 流结束后保存 AI 回复到 SQLite

---

#### `api/screen.py` — 屏幕/Vision 路由（Phase 2）

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/screen/analyze` | POST | 手动截图分析 + AI 点评（SSE 流式） |
| `/api/screen/proactive/start` | POST | 启动主动互动引擎 |
| `/api/screen/proactive/stop` | POST | 停止主动互动引擎 |
| `/api/screen/status` | GET | 查询引擎状态与最近一次结果 |
| `/api/screen/monitors` | GET | 枚举系统显示器 |

**`get_screen_description(app, b64, prompt)` — 核心路由函数：**
- 读取 `get_vision_route(config)` 确定路由
- `multimodal` → 复用 `chat_module` 客户端直接发图片（避免 Key/URL 混用）
- `custom/default` → 调用 `vision_module.analyze_image()`

**`/api/screen/analyze` SSE 事件序列：**
```
{"status": "capturing"}
{"status": "analyzing_vision"}
{"vision_desc": "..."}        ← 视觉描述（可在前端展示）
{"status": "generating"}
{"token": "..."}              ← AI 点评流式 token（多次）
{"done": true, "full_text": "..."}
```

### 核心模块层

#### `modules/core/ai_chat.py` — 文本对话模块

```python
class AIChatModule:
    def __init__(self, config: dict)
    def update_config(self, config: dict)     # 热更新，重建 OpenAI 客户端
    async def chat_stream(messages, system_prompt) -> AsyncGenerator[str, None]
    async def chat_once(messages, system_prompt) -> str
```

**说明：**
- 封装 `AsyncOpenAI` 客户端，支持任意 OpenAI 兼容 API
- `chat_stream` 流式 yield token；遇到异常 yield 错误信息而非抛出
- `chat_once` 收集全部 token 后返回完整字符串

---

#### `modules/core/ai_vision.py` — 视觉路由模块（Phase 2）

**五级路由规则（严格优先级）：**

```
1. 文本模型是多模态（含 gpt-4o / gemini / claude-3 / qwen-vl 等关键词）
   → route = "multimodal"（图片直接嵌入文本模型请求）

2. 非多模态 + custom_vision_enabled = True
   → route = "custom"（使用用户配置的 Vision 端点）

3. 非多模态 + custom_vision_enabled = False
   → route = "default"（硅基流动 Qwen2.5-VL-7B，免费）
```

**多模态模型识别关键词列表：**
`gpt-4o`, `gpt-4-turbo`, `gpt-4-vision`, `claude-3`, `claude-sonnet`, `claude-opus`, `claude-haiku`, `gemini`, `qwen-vl`, `qwen2-vl`, `qwenvl`, `glm-4v`, `llava`, `internvl`, `deepseek-vl`, `minicpm-v`, `vision`, `multimodal`

**Key 分离策略：**
- `default` 路由 → 使用 `vision.siliconflow_key`（专用，避免与文本 Key 混用）
- `custom` 路由 → 使用 `vision.api_key`，为空时降级复用文本模型 Key
- `multimodal` 路由 → 复用 `chat_module` 客户端，本模块不介入

**主要函数：**

```python
is_text_model_multimodal(config) -> bool
get_vision_route(config) -> "multimodal" | "custom" | "default"
build_multimodal_user_message(text, image_b64) -> dict   # 构造多模态 user 消息
get_vision_status(config) -> dict                        # 供前端状态指示器使用

class AIVisionModule:
    async def analyze_image(image_b64, vision_prompt, detail) -> str
```

#### `modules/core/screen_capture.py` — 截图模块（Phase 2）

```python
capture_screen(region, max_width=1280, quality=75) -> (b64: str, width: int, height: int)
list_monitors() -> list[dict]   # 供设置页下拉框使用
```

**说明：**

#### `modules/core/vision_memory.py` — 屏幕记忆模块（Phase 2）

**用途：**
1. 为 AI 对话注入 system prompt（"最近观察到的屏幕变化"）
2. 为主动引擎提供变化检测（Jaccard 相似度判断）

```python
@dataclass
class ScreenObservation:
    timestamp: float
    vision_desc: str
    triggered_by: str     # "proactive" | "user_chat" | "manual"
    associated_reply: str | None
    def formatted_age() -> str    # "30秒前" / "5分钟前"

class VisionMemory:
    def add(vision_desc, triggered_by) -> ScreenObservation
    def get_latest() -> ScreenObservation | None
    def get_recent(n=3) -> list[ScreenObservation]
    def get_context_for_prompt(n=3) -> str    # "[30秒前] 用户在玩CS2..."
    def has_significant_change(new_desc) -> bool   # Jaccard < 0.65 判定为显著变化
    def clear()
```

**设计原则：**
- 纯内存结构（deque），进程重启后清空
- Jaccard 词袋相似度：`|交集| / |并集| < 0.65` 视为有新内容可说

---

#### `modules/core/proactive_engine.py` — 主动互动引擎（Phase 2）

**工作流程（每次 `_tick()`）：**

```
1. 检查用户活跃冷却（默认 60s）→ 跳过
2. 检查最短主动间隔（默认 120s）→ 跳过
3. mss 截图（asyncio.to_thread）
4. 调用 get_screen_description()（统一视觉路由）
5. 变化检测（VisionMemory.has_significant_change）
   └ 无变化连续 4 次 → 强制执行一次（避免永久沉默）
6. AI 自判断：构造含屏幕历史的 prompt → chat_once()
   └ 返回 "[SILENT]" → 不发言
   └ 返回正常文本 → 发言
7. save_message() 存档
8. sio.emit("proactive_message", {...}) 推送前端
```

**AI 自判断 System Prompt 关键部分：**
```
- 当前画面有趣或用户需要帮助 → 主动说话（50~100字）
- 画面平静无变化 → 回复 [SILENT]（仅此四字，不加其他内容）
```

```python
class ProactiveEngine:
    def set_app(app)
    def notify_user_activity()   # 用户发消息时调用，重置冷却计时
    def start() / stop()
    @property is_running -> bool
```

#### `views/VisionPanel.vue` — 游戏陪玩面板（Phase 2）

**观察流设计：**


## 数据流说明

### 普通对话流

```
用户输入
  → ChatPanel.sendMessage()
  → chatStore.sendMessage(text)
  → chatAPI.sendStream()  [POST /api/chat/send]
      → 后端并行截图（若 vision.inject_on_chat）
      → 获取历史 + Prompt
      → 注入视觉信息
      → AIChatModule.chat_stream()
      → SSE token 流
  → onToken 回调 → 累加 aiMsg.content（响应式更新 UI）
  → onDone 回调 → 清除 isStreaming 状态
```

### 主动互动流

```
ProactiveEngine._loop()（后台 asyncio Task）
  → 每 N 秒执行 _tick()
  → 截图 → 视觉分析 → 变化检测
  → AI 自判断（[SILENT] 机制）
  → sio.emit("proactive_message", {...})
      ↓
  Socket.IO → 前端
      ├── ChatPanel: 追加主动消息气泡（青色边框）
      └── VisionPanel: 追加观察记录
```

### 配置更新流

```
用户修改设置
  → SettingsPanel.save()
  → settingsAPI.update(cfg)  [PUT /api/settings]
  → 后端 deep merge → save_config → 热更新各模块客户端
  → 返回新配置
  → settingsStore.config 更新
  → Vue 响应式更新 UI
```
